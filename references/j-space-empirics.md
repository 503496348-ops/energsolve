# J-Space Empirics 增强

> 来源: j-space-skills (J空间认知套件 v3.2)
> 融合目标: 推理卡住时的假设→验证→回写、独立参考、最小复现

## 溺水检测器

计数信号：
- 同一子问题推导两次无新约束
- 规则在无新证据时翻转
- 篇幅增长但预测内容不增
- 实现置信度超过证明质量
- 具体小案例可区分活假设
- 工具调用重复但假设未变
- 开始猜测以保持动量

**两个信号触发实证转向**: 停止纯推导

## 命名未知

每行写一个精确未知：
```
U1: 活动窗口容量是 m-1 还是 m-2？
U2: 不变量是否包含左端点？
```

拒绝模糊未知如"算法令人困惑"。明确哪个变量/假设/边界/因果关系未解决。

## 参数化

每个未知转为有限候选集：
```
CAP ∈ {m-1, m-2}
left_endpoint ∈ {included, excluded}
```

自然空间大/连续时：识别影响决策的维度，选择代表值，说明离散化遗漏。

## 构建独立参考

最慢最简单的方法，避开候选的巧妙假设：
- 穷举枚举
- 直接规格实现
- 手算边界案例
- 已知恒等式/守恒律
- 原始来源
- 改一个变量的最小复现
- 独立推导的预言机

参考不能共享候选的关键捷径。陈述参考自身的限制。

## 设计区分测试

优先强迫候选分歧的测试：
- 空/单例/最小有效案例
- 最大/溢出边界
- 退化/对称案例
- 阈值前后转换

## 证据回写

测试结果写回工作空间：
- 预期与实际匹配→确认候选+记录证据
- 不匹配→排除候选+记录反例
- 不确定→缩小参数空间+设计下一测试

> When theory stops producing distinctions, build a test that can kill a candidate.
